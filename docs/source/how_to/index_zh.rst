任务指南路线图
=====================

当你已经知道自己要完成什么任务，并且需要操作配方而不是学习游览时，从本页开始。全站左侧导航会直接列出每个任务指南页面；
本路线图帮助你选择正确的同级任务页、准备输入，并知道任务页刻意停止时该去哪里查精确事实或设计原因。

页面角色：本页是任务指南（how-to guide）入口，负责把“安装”“检查”“仿真”“生成”“画图”“写模板”“
维护语法资产”等目标分流到真正拥有步骤的页面。

不负责内容：本页不是教程、不是设计解释、不是命令参考，也不是完整字段表。它不替代被链接的任务页，
只说明应该读哪一页以及应期待什么成功信号。

任务指南承诺什么
------------------------

任务指南应该从具体前提开始，给出有序步骤、短输出或明确成功信号，说明常见错误，
并在命令或选项列表变得过宽时链接到参考页。只解释原因的内容属于
:doc:`/explanations/index_zh`；
主要列出精确形式和字段的内容属于 :doc:`/reference/index_zh`。

新用户路径
----------------

如果你刚完成教程，可以这样选择任务页：

1. 快速开始之后，用 :doc:`cli_workflows/index_zh` 把命令链换成自己的输入文件。
2. 语言教程之后，用 :doc:`dsl/index_zh` 修改模型结构。
3. 仿真或检查教程之后，用 :doc:`simulation/index_zh` 或
   :doc:`inspect/index_zh` 处理可重复排查任务。
4. 生成或可视化教程之后，根据目标选择 :doc:`generation/index_zh`、
   :doc:`templates/index_zh` 或 :doc:`visualization/index_zh`。

有经验用户路径
---------------------

如果你已经知道功能领域，按任务动词跳转：

* 准备环境或持续集成任务：打开 :doc:`installation/index_zh`。
* 串联命令流程：打开 :doc:`cli_workflows/index_zh`。
* 修改源语言内容：打开 :doc:`dsl/index_zh`。
* 执行运行行为：打开 :doc:`simulation/index_zh`。
* 读取结构、诊断或大语言模型修复反馈：打开 :doc:`inspect/index_zh`。
* 生成代码或写模板：打开 :doc:`generation/index_zh` 或 :doc:`templates/index_zh`。
* 生成图或维护语法/编辑器资产：打开 :doc:`visualization/index_zh` 或 :doc:`grammar_editor/index_zh`。

维护者路径
----------------

审查任务指南质量时，检查下面几点：

* 每个任务是否有前提、步骤、成功信号、失败边界和下一步链接。
* 示例是否短小并服务于明确目的；长流程是否放在可生成资源里，而不是塞进正文。
* 任务配方是否没有悄悄变成参考表。
* 中文页和英文页是否覆盖相同任务、输出、警告和下一步。

任务页使用信号
---------------------

合格任务页应该让读者在开始前知道自己需要什么输入，
在结束后知道什么现象表示成功。
命令型任务通常需要短输出片段、生成文件名或退出行为；
编辑型任务通常需要指出应该修改哪个文件、
如何检查结果以及常见错误怎样修。

如果任务页只告诉读者“运行这个命令”而不说明成功信号，
它还不够可验收。
如果任务页开始解释大量设计原因，
它应该把原因移交给解释页。
如果任务页列出大量闭合集合选项，
它应该把精确事实移交给参考页。

任务卡片
----------------

安装：:doc:`installation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要在本地、包来源或持续集成镜像里准备 pyfcstm。

读后能力：你能从 PyPI 或主分支安装，验证 Python 包和命令行，并区分核心安装、可选渲染器和原生工具链检查。

不负责内容：它不讲每个命令工作流、语言构造或生成运行时行为。

下一步：安装后运行 :doc:`cli_workflows/index_zh`，
或回到 :doc:`/tutorials/quick_start/index_zh` 跑首次端到端路径。

命令流程：:doc:`cli_workflows/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：已经安装 ``pyfcstm`` 命令，并且有一个 ``.fcstm`` 输入文件或准备复制指南中的小示例。

读后能力：你能在 ``simulate``、``inspect``、``generate``、
``plantuml`` 和 ``visualize`` 之间选择，并识别每类命令的成功信号。

不负责内容：它不列出所有选项、别名、标准输出/标准错误边界或退出状态事实。

下一步：命令事实查 :doc:`/reference/cli/index_zh`；某个流程变深时转到对应功能任务页。

语言任务：:doc:`dsl/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你正在编辑 FCSTM 模型，并知道自己要改状态、转换、事件、守卫、效果、生命周期动作还是导入。

读后能力：你能完成常见建模任务，例如添加状态、编写事件作用域、给转换加守卫、使用生命周期钩子，
并安全选择强制或组合形式。

不负责内容：它不提供每条语法产生式、每个非法形式或完整展开语义。

下一步：精确语法查 :doc:`/reference/dsl/index_zh`；
形式含义不清时看 :doc:`/explanations/dsl_semantics/index_zh`。

仿真任务：:doc:`simulation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：模型能够解析，并且你想执行周期、事件、热启动、抽象处理器或历史导出。

读后能力：你能运行批处理命令、使用交互式命令行、注入事件、热启动到某状态、实现抽象处理器并排查失败模型。

不负责内容：它不完整解释执行顺序模型，也不孤立列出所有接口字段。

下一步：行为原因看 :doc:`/explanations/execution_semantics/index_zh`；
精确命令和接口事实看 :doc:`/reference/simulation/index_zh`。

检查任务：:doc:`inspect/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个 FCSTM 文件，并需要人类报告、JSON 输出、大语言模型修复报告或持续集成严重级别门禁。

读后能力：你能选择报告格式、保存带颜色控制的人类输出、写出完整 JSON、按严重级别让持续集成失败，
并从诊断定位到源文件范围。

不负责内容：它不在正文中定义每个 JSON 字段、每个诊断码或每个静态分析限制。

下一步：报告字段查 :doc:`/reference/inspect_report/index_zh`；
诊断码查 :doc:`/reference/diagnostics_codes/index_zh`；
诊断边界看 :doc:`/explanations/diagnostics/index_zh`。

生成任务：:doc:`generation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：模型能够解析，并且你需要从打包内置模板或自定义模板目录生成产物。

读后能力：你能区分 ``--template`` 和 ``--template-dir``，安全清理输出目录，
生成各内置家族，阅读生成的 README，并做相关烟雾检查。

不负责内容：它不完整讲模板内部，也不会把原生工具链检查设为自动触发。

下一步：模板编写看 :doc:`templates/index_zh`；
内置目标契约查 :doc:`/reference/builtin_templates/index_zh`；
配置键查 :doc:`/reference/template_config/index_zh`。

可视化任务：:doc:`visualization/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个模型，并需要 PlantUML 源码、渲染输出、详细程度预设、聚焦视图、
稳定的持续集成图任务或 Python 接口控制。

读后能力：你能选择源文件或渲染输出、比较预设、渲染最终产物，并让无界面环境里的图任务保持稳定。

不负责内容：它不枚举所有可视化选项或后端失败。

下一步：精确字段看 :doc:`/reference/visualization_options/index_zh`；
需要第一张图路径时回到 :doc:`/tutorials/visualization/index_zh`。

模板编写任务：:doc:`templates/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你已经理解生成基础，并希望创建或调试自定义模板目录。

读后能力：你能建立最小可用模板、理解文件映射、添加表达式和语句样式、注册辅助对象，并验证模板行为。

不负责内容：它不替代精确模板配置参考或内置目标契约。

下一步：配置语法查 :doc:`/reference/template_config/index_zh`；
内置事实查 :doc:`/reference/builtin_templates/index_zh`；
渲染设计看 :doc:`/explanations/template_rendering/index_zh`。

语法和编辑器任务：:doc:`grammar_editor/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你正在修改语法、高亮、VSCode 资产、面向提示的语法指南或验证套件。

读后能力：你能判断语法变更类型，重新生成 ANTLR 产物，同步 Pygments 和 TextMate 高亮，
并验证编辑器扩展资产。

不负责内容：它不完整解释解析器/模型边界，也不替代精确文件和命令参考。

下一步：规范文件查 :doc:`/reference/grammar_tooling/index_zh`；
理解为什么一次语法变更牵涉多个层面时看 :doc:`/explanations/grammar_tooling/index_zh`。

怎样离开任务指南区
---------------------------

任务成功后，需要精确事实就去参考，需要理解意外行为就去解释，需要给队友一个更短的学习路径就去教程。
任务指南应该把这些出口说清楚，而不是同时扮演所有页面角色。
