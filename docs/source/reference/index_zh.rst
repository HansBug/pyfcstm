参考地图
========

当你需要精确事实，而不是学习路径或任务配方时，从本页选择入口。全站左侧导航会直接列出每个参考页，并把自动生成的应用程序接口（API）文档保持为最后的应用程序接口地图；本页帮助你选择正确的查阅目标。

参考页承诺什么
--------------

参考（reference）应该密集且可核查：命令、语法形式、报告字段、选项值、配置键、合法与非法形式，以及指向实现事实的链接。参考不应该变成引导式教程或宽泛设计文章。

按事实类型查阅
--------------

命令名、选项和面向退出结果的行为
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

当你需要命令组、选项名、别名、默认值、输出选择，以及命令失败和模型失败之间的边界时，使用 :doc:`cli/index_zh`。

* **适合在：** 教程或任务指南告诉你需要哪个命令族之后使用。
* **不要期待：** 完整叙事流程；这种内容属于 :doc:`/how_to/cli_workflows/index_zh`。

语言语法和合法形式
~~~~~~~~~~~~~~~~~~

当你需要完整语言查阅时，使用 :doc:`dsl/index_zh`：声明、状态、转换、生命周期块、事件、表达式、注释、不支持形式，以及合法和非法语法例子。

* **适合在：** 你已经知道想写哪个结构之后使用。
* **不要期待：** 完整语义解释；结构为什么这样行为，属于 :doc:`/explanations/dsl_semantics/index_zh`。

检查报告字段
~~~~~~~~~~~~

当你需要检查输出的形状和含义，包括面向人类和面向机器的报告字段时，使用 :doc:`inspect_report/index_zh`。

* **适合在：** ``pyfcstm inspect`` 已经产出报告，或教程展示过报告族之后使用。
* **不要期待：** 每个诊断码解释；诊断码查阅属于 :doc:`diagnostics_codes/index_zh`。

诊断码
~~~~~~

当你需要诊断标识符、严重度、信息含义和修复方向时，使用 :doc:`diagnostics_codes/index_zh`。

* **适合在：** 检查、验证或命令信息给出诊断码之后使用。
* **不要期待：** 泛化建模建议；上下文属于语言解释和诊断解释。

仿真命令、运行时设置和 Python 应用程序接口事实
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

当你需要仿真器命令形式、交互命令参数、事件输入形式、设置、历史/导出形状、Python 运行时应用程序接口事实、``CycleResult`` 字段和公开失败边界时，使用 :doc:`simulation/index_zh`。

* **适合在：** 仿真教程或任务指南已经展示工作流，而你需要精确值或边界情况之后使用。
* **不要期待：** 执行顺序理由；语义推理属于 :doc:`/explanations/execution_semantics/index_zh`。

可视化选项
~~~~~~~~~~

当你需要精确渲染选项、输出格式行为、渲染器假设和图导出边界条件时，使用 :doc:`visualization_options/index_zh`。

* **适合在：** 你已经知道自己需要 PlantUML 源码还是渲染图之后使用。
* **不要期待：** 第一张图的引导流程；这种内容属于 :doc:`/tutorials/visualization/index_zh`。

模板配置和内置模板事实
~~~~~~~~~~~~~~~~~~~~~~

使用 :doc:`template_config/index_zh` 查渲染器配置键、对象加载形式、表达式与语句风格设置，以及非法配置形状。使用 :doc:`builtin_templates/index_zh` 查打包模板名、状态、生成产物和目标语言族边界。

* **适合在：** 你正在生成代码或维护模板时使用。
* **不要期待：** 超出模板参考负责范围的目标运行时语义；流程和设计背景属于生成任务指南与模板解释。

语法和编辑器工具事实
~~~~~~~~~~~~~~~~~~~~

当你需要精确语法文件、生成解析器路径、高亮资产、编辑器支持资产和验证命令时，使用 :doc:`grammar_tooling/index_zh`。

* **适合在：** 计划修改语法或编辑器支持之前使用。
* **不要期待：** 为什么这些资产必须耦合的设计原因；这种内容属于 :doc:`/explanations/grammar_tooling/index_zh`。

自动生成的 Python 应用程序接口地图
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

当你需要完整自动生成的 Python 应用程序接口结构时，使用 :doc:`应用程序接口文档 </api_doc_zh>`。自动生成的应用程序接口地图保留在参考区最后一项；如果需要修改应用程序接口页面引导，应该改生成脚本，而不是直接编辑生成输出。
