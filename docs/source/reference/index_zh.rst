参考地图
================

当你需要查准事实，而不是学习路线、任务配方或设计解释时，从本页开始。全站左侧导航会直接列出每个参考页面，
并把自动生成的应用程序接口（API）文档保留为最后的 API 地图；本页按事实类型帮助你选择查阅目标。

页面角色：本页是参考（reference）入口，指向命令、语言语法、检查报告字段、诊断码、仿真事实、可视化选项、
模板配置、语法工具链、内置模板和生成 Python API 对象的密集查阅页。

不负责内容：本页不是教程，不是任务步骤，也不是概念长文。它不重复每个参考页里的完整表格，只说明查阅边界。

参考页承诺什么
---------------------

参考页应该准确、可搜索，并对自己声称覆盖的事实形成闭合集合。它应该包含精确拼写、默认值、合法值、非法形式、字段含义、
副作用、诊断，以及必要时指向事实源。读者不应该必须先完成教程才能查一个值。

新用户路径
----------------

如果你刚接触项目但已经遇到查阅问题，先按事实家族选择：

1. 查命令选项或输出行为，看 :doc:`cli/index_zh`。
2. 查语法形式，看 :doc:`dsl/index_zh`。
3. 查检查 JSON 或人类报告字段，看 :doc:`inspect_report/index_zh`。
4. 查诊断标识和修复方向，看 :doc:`diagnostics_codes/index_zh`。
5. 查 Python 对象、类、函数或模块时，在知道对象名之后看 :doc:`/api_doc_zh`。

有经验用户路径
---------------------

如果你已经知道子系统，直接查最短入口：

* 面向命令的行为：:doc:`cli/index_zh`。
* 语言形式：:doc:`dsl/index_zh`。
* 报告形状：:doc:`inspect_report/index_zh`。
* 诊断码：:doc:`diagnostics_codes/index_zh`。
* 运行时执行接口：:doc:`simulation/index_zh`。
* 图选项字段：:doc:`visualization_options/index_zh`。
* 模板配置键或内置目标：:doc:`template_config/index_zh` 或 :doc:`builtin_templates/index_zh`。
* 语法/编辑器维护文件或命令：:doc:`grammar_tooling/index_zh`。
* 公共 Python API 对象：:doc:`/api_doc_zh`。

维护者路径
----------------

审查参考质量时，检查下面几点：

* 每个参考页是否为闭合集合提供表格或结构化事实。
* 复杂项目是否包含合法例子、非法例子和失败边界。
* 面向命令的行是否在需要时说明标准输出、标准错误、退出状态或文件副作用。
* 自动生成的 API 页面是否仍由生成流程维护，而不是被手写参考正文替代。
* 中英文参考页是否暴露相同事实、默认值、例子和风险边界。

查阅卡片
----------------

命令：:doc:`cli/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你知道要查哪个命令家族，或需要在 ``simulate``、``inspect``、
``generate``、``plantuml`` 和 ``visualize`` 之间选择。

读后能力：你能查命令名、选项名、别名、默认值、可接受值、输出行为、例子和面向命令的失败边界。

不负责内容：它不教授完整工作流，也不解释底层模型为什么这样行为。

下一步：任务配方看 :doc:`/how_to/cli_workflows/index_zh`；
命令输出让你意外时看对应功能解释。

FBMCQ 查询语言：:doc:`bmc_query/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要精确查询初态、假设、谓词、调用过滤器、性质、默认值或非法形式。

读后能力：你能查完整语言和三层合法性；每个高影响家族都有多组合法例子与明确边界。

不负责内容：它不解释求解器数学原理，也不教授第一次检查。

下一步：操作任务看 :doc:`/how_to/bmc/index_zh`；目标语义看
:doc:`/explanations/bmc_properties/index_zh`。

有界模型检查命令与结果协议：:doc:`bmc_results/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要精确命令、输出流、颜色、退出状态、JSON、见证、重放、耗时、诊断或打包事实。

读后能力：你能区分性质结论与 SAT/UNSAT，消费版本化 JSON 模式，并处理所有有报告分支与
错误分支。

不负责内容：它不定义 FBMCQ 语法，也不推导有界公式。

下一步：查询形式查 :doc:`bmc_query/index_zh`；求解与重放原理看
:doc:`/explanations/bmc_solving/index_zh`。

语言语法：:doc:`dsl/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你知道要查哪个 FCSTM 构造，例如声明、导入、状态、转换、事件、表达式、生命周期块、强制形式、
组合形式或注释。

读后能力：你能查合法语法、非法反例、展开准规范、表达式优先级、操作块、事件作用域和相关诊断。

不负责内容：它不提供第一条建模路径，也不完整解释语义动机。

下一步：建模任务看 :doc:`/how_to/dsl/index_zh`；
语义看 :doc:`/explanations/dsl_semantics/index_zh`。

检查报告字段：:doc:`inspect_report/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你运行或准备运行 ``pyfcstm inspect``，并需要查人类输出、
JSON 输出或大语言模型输出的形状。

读后能力：你能查影响报告的命令选项、顶层 JSON 字段、嵌套对象契约、大语言模型报告契约和非法输入边界。

不负责内容：它不列出每个诊断标识，也不解释诊断理念。

下一步：诊断码查 :doc:`diagnostics_codes/index_zh`；
任务看 :doc:`/how_to/inspect/index_zh`；
能力边界看 :doc:`/explanations/diagnostics/index_zh`。

诊断码：:doc:`diagnostics_codes/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你从检查输出里拿到了诊断标识、严重级别或修复提示，并需要查它的含义。

读后能力：你能阅读诊断家族、输出层级、消息含义、修复方向和机器覆盖检查。

不负责内容：它不定义完整检查报告模式，也不教授排查工作流。

下一步：报告字段查 :doc:`inspect_report/index_zh`；
可重复排查任务看 :doc:`/how_to/inspect/index_zh`。

仿真事实：:doc:`simulation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要精确的仿真命令调用、批处理或交互命令形式、事件输入形式、设置、
历史/导出事实或 Python 运行时接口事实。

读后能力：你能查命令语法、设置名、可接受值、历史格式、公开失败和运行时接口边界。

不负责内容：它不解释完整周期顺序推理，也不教授第一次仿真运行。

下一步：任务看 :doc:`/how_to/simulation/index_zh`；
执行原因看 :doc:`/explanations/execution_semantics/index_zh`。

可视化选项：:doc:`visualization_options/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要精确的图选项字段、渲染器行为、详细程度预设、后缀处理、无界面行为或失败边界。

读后能力：你能查选项场景、PlantUML 选项字段、渲染器示例卡片、预设解析、变量/状态标签、
生命周期可见性以及转换/事件选项。

不负责内容：它不教授第一张图工作流，也不把每张图写成学习叙事。

下一步：任务看 :doc:`/how_to/visualization/index_zh`；
第一张图看 :doc:`/tutorials/visualization/index_zh`。

模板配置：:doc:`template_config/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你正在写或调试模板目录，并需要查精确的 ``config.yaml`` 键、对象加载形式、映射规则、
忽略行为或渲染辅助对象事实。

读后能力：你能查表达式样式、语句样式、运行时语句渲染器、Jinja 辅助对象、验证失败、文件映射和内置配置例子。

不负责内容：它不从原理上解释渲染流水线，也不列出每个内置模板契约。

下一步：编写任务看 :doc:`/how_to/templates/index_zh`；
设计看 :doc:`/explanations/template_rendering/index_zh`；
内置目标事实看 :doc:`builtin_templates/index_zh`。

语法和编辑器工具：:doc:`grammar_tooling/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要规范语法文件、生成解析器边界、高亮事实、VSCode 资产、运算符顺序、关键字更新步骤或验证命令事实。

读后能力：你能查源文件/生成文件配对、核心维护命令、Pygments 和 TextMate 事实、
VSCode 验证套件和更新清单。

不负责内容：它不解释为什么语法变更会扩散到多个层面，也不教授初学者语法路线。

下一步：任务看 :doc:`/how_to/grammar_editor/index_zh`；
原因看 :doc:`/explanations/grammar_tooling/index_zh`。

内置模板：:doc:`builtin_templates/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你需要知道哪些打包目标模板存在、如何发现它们、生成 README 承诺什么，或每个目标语言族需要什么。

读后能力：你能查元数据、发现接口、生成 README 契约，以及 ``python``、``c``、
``c_poll``、``cpp`` 和 ``cpp_poll`` 的目标族说明。

不负责内容：它不教授自定义模板编写，也不详细解释渲染器架构。

下一步：生成任务看 :doc:`/how_to/generation/index_zh`；
模板编写看 :doc:`/how_to/templates/index_zh`；
配置事实查 :doc:`template_config/index_zh`。

生成 API 地图：:doc:`/api_doc_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你已经知道要查的 Python 模块、类、函数或数据对象，或某个参考页已经指向了对应 API 对象。

读后能力：你能浏览根据 ``pyfcstm/`` 生成的模块树，并查公共 Python API 文档字符串。

不负责内容：它不替代命令、语言、报告、模板或可视化参考页，也不应该手工编辑。

下一步：如果还不知道 API 对象名，请先用功能参考页；如果 API 参考页导言需要改变，应修改生成脚本。

如果你需要其他页面角色
---------------------------------

使用解释。
