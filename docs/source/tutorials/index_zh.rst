教程路线图
================

第一次学习 pyfcstm，并且希望每读完一页都有可见进展时，从本页开始。全站左侧导航会直接列出每个教程；
本路线图只是同级页面的阅读指南，用来说明下一页该读什么、为什么读它，以及当教程刻意停在入门边界时应该去哪里继续。

页面角色：本页是教程（tutorial）入口，负责安排首次成功路径。它帮助读者先看到一个能运行、能检查、能生成、
能画图的最小闭环，而不是一上来阅读完整参考。

不负责内容：本页不负责完整语法、命令选项、检查报告字段、诊断码、模板配置或应用程序接口（API）地图。
需要精确事实时看 :doc:`/reference/index_zh`；
已经知道任务时看 :doc:`/how_to/index_zh`；
想理解原因时看 :doc:`/explanations/index_zh`。

怎样阅读教程
------------------

教程负责首次成功。合格教程应该给出一个输入、一条短命令或短代码、可观察输出，以及下一步链接。它不会把所有选项、
所有非法形式或所有诊断码塞进学习路径。

当你的问题从“第一条可运行路径是什么”变成“我怎样在项目里重复这件事”或“这个形式的精确定义是什么”时，
就应该离开教程，转到任务指南、解释或参考页。

新用户路径
----------------

如果你刚接触 pyfcstm，按下面顺序读：

1. 从 :doc:`quick_start/index_zh` 开始，确认一个小文件可以被仿真、检查、生成和导出图。
2. 读 :doc:`dsl/index_zh`，理解第一个源文件为什么长成这样。
3. 选择第一个反馈循环：想看运行行为就读 :doc:`simulation/index_zh`，
   想看结构和诊断反馈就读 :doc:`inspect/index_zh`。
4. 想从同一个模型产出代码时读 :doc:`generation/index_zh`。
5. 想用图向别人解释模型结构时读 :doc:`visualization/index_zh`。
6. 最后把 :doc:`bmc/index_zh` 作为高级收官。此时你已经能读懂 FCSTM 模型和一条
   具体运行轨迹，再学习怎样把一组手工事件替换为对有限范围内全部允许事件序列的符号搜索。

有经验用户路径
---------------------

如果你已经熟悉项目，只需要找回某个工作流：

* 已有 FCSTM 文件但不确定模型是否可靠：跳到 :doc:`inspect/index_zh`，再查 :doc:`/reference/diagnostics_codes/index_zh`。
* 模型合法但需要行为证据：跳到 :doc:`simulation/index_zh`，再看 :doc:`/how_to/simulation/index_zh`。
* 已经理解模型，只需要输出文件：跳到 :doc:`generation/index_zh` 或 :doc:`visualization/index_zh`。
* 已经理解具体轨迹，需要检查有限边界内的全部行为：最后读 :doc:`bmc/index_zh`。
* 需要精确语法、命令选项或报告字段：直接从 :doc:`/reference/index_zh` 开始。

维护者路径
----------------

审查或扩展教程时，按下面清单看：

* 每篇教程是否从真实初学者状态开始，并且只有一个主要成功路径。
* 命令和代码是否短小，是否展示了可观察结果，而不是把长脚本塞进正文。
* 教程是否说明自己在哪里停止，并把更深内容交给任务指南、解释或参考页。
* 中英文页面是否讲同一能力、同一风险边界和同一下一步。

读者完成信号
------------------

读完一个教程后，应该能回答三个问题。

* 我刚刚运行或阅读的输入文件是什么。
* 我看到的输出、状态变化、报告或图说明了什么。
* 如果要重复、查准或解释这件事，应该离开教程去哪里。

如果这些问题答不上来，教程正文需要补充更短的输出片段、
更清晰的结果解释，或更明确的下一步链接。
如果这些问题已经能回答，就不要继续把参考表搬进教程，
而应该把读者送到对应任务指南、解释或参考页。
教程可以短小，但必须让读者看见自己完成了哪一环，
以及下一种页面角色会接住什么内容。

同级教程卡片
------------------

快速开始：:doc:`quick_start/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：已经安装或准备先按 :doc:`/how_to/installation/index_zh` 安装
``pyfcstm`` 命令。

读后能力：你能在一个交通灯模型上看到仿真、检查、生成和 PlantUML 输出属于哪些家族。

不负责内容：它不解释每个命令选项、每个生成文件，也不解释状态机执行语义。

下一步：想理解输入文件就读 :doc:`dsl/index_zh`；
只想重复命令流程就读 :doc:`/how_to/cli_workflows/index_zh`。

语言教程：:doc:`dsl/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：已经在快速开始里见过一个小 FCSTM 文件，并且可以编辑文本文件。

读后能力：你能写出包含状态、转换、守卫、生命周期动作和一次修复练习的小模型。

不负责内容：它不是完整语法参考；合法形式、非法形式、语法糖展开和诊断归
:doc:`/reference/dsl/index_zh`。

下一步：想执行模型就读 :doc:`simulation/index_zh`；
想理解构造含义就读 :doc:`/explanations/dsl_semantics/index_zh`。

仿真教程：:doc:`simulation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个合法示例模型，并且想观察活跃状态变化，而不只是确认文本能解析。

读后能力：你能运行批处理记录或短 Python 循环，并读懂周期或事件之后的活跃路径。

不负责内容：它不枚举所有交互命令、显示设置、历史格式或热启动边界。

下一步：重复任务看 :doc:`/how_to/simulation/index_zh`；
精确命令和接口事实看 :doc:`/reference/simulation/index_zh`。

检查教程：:doc:`inspect/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个模型，并希望 pyfcstm 说明它读到了什么、推断了什么、哪些诊断能指导修复。

读后能力：你能生成面向人类、JSON 和大语言模型的检查输出，并理解诊断怎样指回源文件位置。

不负责内容：它不列出所有 JSON 字段、诊断码、严重级别或可选验证边界。

下一步：排查任务看 :doc:`/how_to/inspect/index_zh`，
字段和诊断码查 :doc:`/reference/inspect_report/index_zh` 与
:doc:`/reference/diagnostics_codes/index_zh`。

生成教程：:doc:`generation/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个合法模型，并希望用打包内置 Python 模板生成文件，而不是先学习模板内部。

读后能力：你能运行 ``pyfcstm generate --template python``，
并查看第一批生成的运行时产物。

不负责内容：它不讲自定义模板编写、每个内置目标契约、格式化策略或渲染器内部。

下一步：目标生成任务看 :doc:`/how_to/generation/index_zh`；
模板编写看 :doc:`/how_to/templates/index_zh`；
渲染流水线看 :doc:`/explanations/template_rendering/index_zh`。

可视化教程：:doc:`visualization/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：你有一个合法模型，并希望得到能向别人解释结构的图源码或渲染图。

读后能力：你能导出 PlantUML 源码，理解示例图证明了什么，并尝试第一组详细程度预设。

不负责内容：它不列出所有可视化选项、渲染后端、后缀规则或无界面环境边界。

下一步：图任务看 :doc:`/how_to/visualization/index_zh`；
精确选项行为查 :doc:`/reference/visualization_options/index_zh`。

有界模型检查：:doc:`bmc/index_zh`
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

读前条件：已经完成语言教程，并且能从仿真或检查结果中读懂至少一条具体轨迹；知道一个周期既可能改变
活跃状态，也可能更新持久变量。

读后能力：你能把一条手工选择的执行替换为有界符号搜索，写出有实际意义的安全性质，读懂反例轨迹，
修复模型，并说明为什么修复后的有界结果仍不是无界证明。

不负责内容：它不枚举 FBMCQ 文法，也不推导每一条转换关系和性质目标。

下一步：重复检查任务看 :doc:`/how_to/bmc/index_zh`；数学原理从
:doc:`/explanations/bmc_semantics/index_zh` 开始；精确查询形式查
:doc:`/reference/bmc_query/index_zh`。

教程路径在哪里停止
---------------------------

需要广度或精度时，请主动离开教程。重复操作属于任务指南，设计原因属于解释，精确事实属于参考。这样教程能保持短小，
而每个能力仍有更深的归属页。
